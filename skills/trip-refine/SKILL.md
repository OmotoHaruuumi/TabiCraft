# trip-refine — プラン選定＆対話修正（Phase 3）

選ばれた案を、実際に動ける詳細旅程へ。**叩き台を一緒に育てる**フェーズ。分単位の現実性をここで担保する。
出力は旅行フォルダの `plans/<案名>.md`。

## 入力
- `combinations.md` / `trip-basics.md` / `~/.claude/travel/profiles/*.md`

## 入力モード（最初に確認）
着手前に AskUserQuestion で進め方を聞く：**🙋自分で決める / ✨おすすめにまかせる / 🤝一緒に決める（既定） / 🛑一度止めて相談する**。
**🛑「一度止めて相談する」は、このスキルで選択肢を出すすべての場面で必ず付ける**（選ばれたら自動進行を止め会話する）。
- 🙋自分で決める：ユーザーが各日のスポット・時刻を指定 → こちらは移動時間の実測・営業時間・終電/乗継・予算の裏取りに徹する。
- ✨おすすめにまかせる：選んだ案をこちらが詳細タイムラインまで一気に組み、ユーザーは確認・微修正のみ。
- 項目（日・スポット）ごとに混在OK。

## 手順

### 1. 案を選ぶ
- どの案を詰めるか確認（複数並行で詰めて比較してもよい）。
- 選んだ案を `plans/<案名>.md` に展開（combinations の日割り見出しを骨格にする）。

### 2. 詳細化（Day別タイムライン）
各日を「朝→午前→昼食→午後→夕食→夜（宿へ）」のフレームで埋める。各行に:
- 時間帯 / やること・目的地 / メモ（入場料・予約要否・所要時間）/ 移動（手段・所要）

**Day1 は到着・最終日は帰宅から始める**（trip-basics の到着帰宅パターンを反映）。
毎日の最終行は「🏨 <宿名> へ戻る」で締める（宿名を必ず入れる）。

### 3. 現実性チェック（このフェーズの肝・他スキルから採用）
- **移動時間の実測検証**: 各区間を WebSearch で実測確認する。LLMの直感は30〜50%楽観的になりがちなので必ず裏を取る。詰め込みすぎなら警告して案を出す。
- **営業時間・定休日を旅行日で確認**: 休館日の地雷を潰す。
- **終電・最終便・乗継リスクの注記**: 到着が遅い/乗継が必要な区間に「末班〇〇:〇〇、△時前到着推奨」を明記。
- **地理クラスタリング**: 同エリアを同日にまとめ、無駄な往復を消す。
- **ペース**: プロファイルに合わせ激務日と休息日を交互に。のんびり派なら1日2〜3スポットに抑える。

### 余裕とオプションを散りばめる（旅はハプニングの連続・最重要の思想）
旅は遅延・道迷い・気分の変化・軌道修正がつきもの。詰め込まず、立て直せる余白を最初から組み込む。
- **各スポットに移動＋15〜30分のバッファ**を持たせ、1日の予定は「行けたら全部」ではなく**8割で組む**。
- 1日の**必須は2〜3個に絞る**。残りは **⏳「時間が余ったらコレ」候補**として近くに散りばめる（回れなくても気にしない前提のオプション）。
- 移動後や昼下がりに **🪶 ゆとり枠（カフェ・休憩・自由時間）** を意図的に置く。
- 雨・疲れ・休館・行列に備え、各日に**屋内/近場の代替案を1つ**用意。
- 旅程には「必須」と「⏳余ったら」を**見分けられるよう区別**して書き、しおり（trip-brochure）へ引き継ぐ。

### 4. 予算（信頼度3段階で正直に）
交通・宿・食・アクティビティ・現地交通・予備費に分け、各金額に信頼度を付ける:
- 実査（確定価格・標注なし）／ 参考 `~`（検索の相場）／ 推定 `≈`
- 海外なら日本円換算を併記。末尾に「出発前に最新価格を要確認」と一文。

### 5. 対話で修正（中心作業）
- ユーザーの「ここ入れ替えたい/詰めすぎ/もっと贅沢に/予算オーバー」に応じて Edit で更新。
- 代替案を出すときは理由を添える（天気・体力・予算・衛生）。
- 修正中に出た好み発言は trip-profile モードB で観察ログに追記（学習ループ）。
- 旅程は叩き台。一緒に育てる姿勢で。

### 6. 完成確認 → 次フェーズ
ユーザーが「OK／これで確定」と言ったら、その plan md を確定版とし:
> `plans/<案名>.md` を確定版にしました。
> かっこいい HTML の旅のしおりにするなら `trip-brochure` を実行してください。

## plans/<案名>.md の構成
```
# <案名>（確定 / 検討中）
## 概要（ルート・泊数・メンバー・宿）
## Day1 … DayN（時間帯テーブル：時間 / やること / メモ / 移動。必須は2〜3個に絞り、⏳余ったら候補・🪶ゆとり枠・☔代替案を併記）
## 予算内訳（信頼度つき）
## 現実性メモ（移動実測・終電注記・定休日・天気代替・バッファ確保）
## 持ち物・準備リスト
```

## 注意
- 宿名は trip-combine/refine で確定したものを固定し、以降ブレさせない。
- 「ホテルへ戻る」とだけ書かず必ず宿名を入れる。
- 多都市なら都市ごとに宿名を切り替えて記載。
