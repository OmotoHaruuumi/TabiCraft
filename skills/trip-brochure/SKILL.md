# trip-brochure — 旅のしおり HTML 生成（Phase 4）

確定プランを、見て楽しく実際に使える**単一ファイルHTMLのしおり**にする。
出力は旅行フォルダの `output/<旅行名>.html`（CSS/JSは内部化・1ファイル完結）。

## 入力
- `plans/<案名>.md`（確定版。複数あればどれか確認）
- `trip-basics.md`（前提・通貨・季節メモ）
- 参考テンプレート: スキルフォルダ内 `assets/brochure-template.html`

## 入力モード（最初に確認）
着手前に AskUserQuestion でデザインの決め方を聞く：**🙋自分で指定（配色・載せる情報・トーンを指示） / ✨おまかせ（行き先の雰囲気でこちらが最適化） / 🛑一度止めて相談する**。
**🛑「一度止めて相談する」は、このスキルで選択肢を出すすべての場面で必ず付ける**（選ばれたら自動進行を止め会話する）。
- 自分で指定なら、その希望（色・順序・載せる/省く項目）を反映。
- おまかせなら行き先の雰囲気で配色・装飾を自動決定。

## 手順

### 1. データ抽出
plan md と basics から、しおりに載せる要素を構造化して取り出す:
- 旅行名・期間・メンバー・ルート / 宿（都市ごと）/ Day別タイムライン / 移動手段 / 予算内訳 / 持ち物・準備 / 現地メモ（気候・通貨・注意）

### 2. Google Maps 経路リンク生成（各移動行）
形式:
`https://www.google.com/maps/dir/?api=1&origin=<FROM>&destination=<TO>&travelmode=<MODE>`
- travelmode: transit（電車地下鉄バス・空港送迎）/ walking（徒歩15分以内）/ driving（郊外）/ bicycling（広い公園史跡）
- 場所名は英語で（例: `Wat+Pho+Bangkok`）。
- **各日の朝の origin は必ず宿名**（エリア名・駅名はNG）。例: `origin=Riva+Surya+Bangkok`。
- 認証不要のただのURL。外部連携ではないので「HTMLのみ」方針と矛盾しない。不要ならこのステップを省ける。

### 3. HTML 生成
`assets/brochure-template.html` を土台に、抽出データを流し込む。品質要件:
- **単一ファイル**（CSS/JS内部化・画像はCDN絵文字や軽量アイコンで代替）。
- **レスポンシブ**（スマホで旅行中に見る前提。タップしやすいボタン）。
- **行き先の雰囲気で配色・装飾を変える**（南国＝暖色＋海、欧州都市＝モノトーン上品 など。毎回同じにしない）。
- 構成: ヒーロー（旅行名・日付・ルート）→ 概要カード（宿/通貨/移動アプリ/注意）→ 予算サマリー→ Day別タイムライン（Mapsボタン付き）→ 持ち物・準備チェック→ 現地メモ。
- 予算は信頼度（実査/参考~/推定≈）を視覚的に区別。
- 日本語UI。温かみのある旅好きの友人のトーン。
- 生成後、HTMLが壊れていないか（タグ閉じ・JS構文）をざっと自己チェック。

### 4. 保存・共有
- `output/<旅行名>.html` に Write。
- パスを伝え、ブラウザで開けば完成形が見られると案内。
> 旅のしおりを `output/<旅行名>.html` に作りました！ブラウザで開いて確認してください。
> 直したい所（配色・載せる情報・予定）があれば言ってください。Mapsリンクが不要なら外せます。

### 4.5 PDF / PNG で保存（求められたら）
しおりは HTML だけでなく PDF・PNG でも残せる。AskUserQuestion で「HTMLのまま / PDFで保存 / PNGで保存 / 🛑一度止めて相談する」を提示。
- HTML を開いて画面右上の **🖨 PDFで保存** ボタン（`window.print()`）→ ブラウザの「PDFとして保存」でも出せる（最も確実・誰でも可）。
- スクリプトで一括書き出しもできる。Bash で:
  - PDF: `~/.claude/skills/trip-brochure/export.sh output/<旅行名>.html output/<旅行名>.pdf`
  - PNG: `~/.claude/skills/trip-brochure/export.sh output/<旅行名>.html output/<旅行名>.png`
  （Google Chrome / Chromium / Edge のヘッドレスを利用。見つからなければブラウザ印刷を案内する。）
- 出力先は `output/` 配下（gitignore 済みなので公開されない）。

### 5. 学習ループ（締め）
しおり完成は1旅行の区切り。気に入った点・反省点が会話に出たら trip-profile モードB で各人の観察ログに追記し、次回の提案精度を上げる。

## 任意の拡張（求められたら）
- Leaflet + CARTO の軽量インタラクティブ地図埋め込み（オフライン耐性・iframe不使用）。
- 印刷/PDF向けレイアウト。
- 多言語併記（現地語の指さしフレーズ）。

## 注意
- plan が未確定なら refine に戻す（確定前にしおり化しない）。
- 宿名・時刻は plan md と完全一致させる（勝手に変えない）。
