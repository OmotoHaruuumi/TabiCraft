# trip-basics — 基本情報の確定（Phase 1）

旅程づくりの骨格を固める。推測で進めず、未定の項目は複数パターンを出してユーザーに選ばせる。
出力は旅行フォルダ（カレントディレクトリ）の `trip-basics.md`。

## 入力
- 旅行者プロファイル: `~/.claude/travel/profiles/*.md`（あれば全員ぶん Read。無ければ trip-profile を案内）

## 入力モード（最初に確認）
着手前に AskUserQuestion で進め方を聞く：**自分で決める / おすすめにまかせる / 一緒に決める（既定） / 一度止めて相談する**。項目ごと混在OK。
**「一度止めて相談する」は、このスキルで選択肢を出すすべての場面で必ず付ける**（選ばれたら自動進行を止め会話する）。
- 自分で決める項目（例: 行き先が決まっている）は、その内容をそのまま採用し候補を押し付けない。下調べ（気候・季節の罠等）だけ補助する。
- おすすめの項目は、プロファイル＋WebSearch で候補を出して選んでもらう。

## 手順

### 1. プロファイル読込
- 同行者ぶんのプロファイルを Read し、雰囲気・ペース・予算・苦手を把握。
- 学習ループ: この会話で好みを示す発言が出たら `trip-profile` モードB で観察ログに追記する。

### 2. 基本情報ヒアリング（AskUserQuestion・1回2問まで）
埋める項目:
- **人数 / 誰と**（一人 / カップル / 家族 / 友人グループ）
- **行き先**（決まっていれば具体地名。未定なら → 後述の複数パターン）
- **泊数 / 時期**（何泊・いつ頃。時期未定なら気候・料金からおすすめ時期を提示）
- **到着の時間帯と場所**（例: 朝便で〇〇空港 / 夜便 など）
- **帰宅の時間帯と場所**

### 3. 未定項目は「複数パターン」を出す（重要）
決まっていない項目は1つに決めつけず、選べる形で提示する。
- **行き先未定**: プロファイルの雰囲気＋泊数＋予算で WebSearch し、3〜5候補を表で提示（行き先 / おすすめ理由 / 費用目安 / 移動時間 / ベスト時期）。AskUserQuestion で選択。
- **到着/帰宅の時間帯未定**: 代表パターンを2〜3案。各案で初日・最終日の使える時間がどう変わるかを一言添える。
  - 例: 「① 朝着便 → 初日丸一日使える / ② 夜着便 → 初日は移動と食事のみ・宿近辺で完結」
- パターンは `trip-basics.md` に**全て残す**（後で trip-combine が分岐を考慮できるように）。

### 4. 下調べ（WebSearch、国内外バランス対応）
行き先が決まったら必ず調べて `trip-basics.md` に記載:
- **時期の気候・混雑・イベント**（雨季 / 祭り / 連休 / 暑さ寒さ / 観光ハイシーズン）
- **季節の罠**（閉鎖期・枯水期・禁漁期・特定施設の休業期間など、行ってがっかりする要素）
- **海外の場合**: ビザ / 入国要件 / 通貨と日本円換算レート / 現地の主流配車アプリ（東南アジア=Grab、欧州=Bolt/Uber 等）/ 治安・衛生の具体的注意
- **国内の場合**: 主要な移動手段（新幹線/在来線/レンタカー要否）/ 繁忙期の予約逼迫
- 到着/帰宅が空港・駅なら、市内中心部までの所要時間と手段も調べておく。

### 5. 保存
`trip-basics.md` を Write。構成:
```
# 基本情報：<旅行名>
## メンバー（人数・誰と・各自プロファイルへのリンク）
## 行き先・ルート（確定 or 候補パターン）
## 期間・時期（泊数・日付・ベスト時期の根拠）
## 到着 / 帰宅（時間帯・場所。未定なら全パターン）
## 現地の前提（気候・季節の罠・ビザ・通貨・配車アプリ・治安衛生）
## 予算レンジ（ざっくり総額・1日目安）
```

### 6. 次フェーズへ案内
> 基本情報を `trip-basics.md` に保存しました。
> 次は「行きたい場所・食べたいもの・宿の条件」を集めて組み合わせを作ります。

このとき、旅行フォルダに `wishlist.md` が無ければ、共有マスター
`~/.claude/travel/templates/wishlist-template.md` を旅行フォルダに `wishlist.md` としてコピーして用意する。
そのうえでユーザーに2択を示す:
- **自分で書く**: `wishlist.md` を開いて記入してもらう。
- **対話で埋める**: AskUserQuestion で「行きたい所/食べたい物/宿の条件/備考」を聞き取り、こちらが `wishlist.md` に書き込む（テンプレ編集が面倒な人向け・推奨）。

## 注意
- 未定を勝手に確定しない。必ず選択肢として残す。
- ここでホテルや具体スポットを1つに絞り込まない（それは combine / refine の仕事）。
- **成果物（trip-basics.md）には最新の確定形だけを残す**（変更履歴・改訂理由・旧版メモは書かない。訂正が入ったら古い記述は上書き/削除し、常に「今の前提」だけにする。経緯はチャットで説明）。
