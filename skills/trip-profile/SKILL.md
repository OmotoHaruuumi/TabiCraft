# trip-profile — 旅行者プロファイル＆学習エンジン（Phase 0）

旅行者ごとの好みを**一度きりのアンケートではなく、使うほど賢くなる学習ファイル**として育てる。
同行者が「〜がしたい」「〜は苦手」と言うたびに観察ログへ追記し、安定した傾向を蒸留する。
他の全フェーズ（basics / combine / refine / brochure）はこのファイルを読み、提案を本人に最適化する。

## 保存場所
- プロファイル: `~/.claude/travel/profiles/<名前>.md`（旅行をまたいで永続・再利用）
- テンプレート（共有マスター）: `~/.claude/travel/templates/profile-template.md`

## プロファイルの構造（3層）
1. **確定プロファイル** — 安定した好み（蒸留済み・提案で直接使う）
2. **観察ログ（追記専用）** — 発言・行動の生の記録。各エントリに「日付 / 文脈 / 発言や行動 / そこから読み取った好み」
3. **気づいた傾向** — 観察ログから合成した、本人が言語化していない傾向

## 3つの起動モード

### モード A: 新規作成（ファイルが無い）
1. `~/.claude/travel/profiles/` を Glob で確認。
2. AskUserQuestion で進め方を聞く：**自分で書く（テンプレに記入） / 対話で埋める（質問に答えるだけ・既定） / 一度止めて相談する**。
   **「一度止めて相談する」は、このスキルで選択肢を出すすべての場面で必ず付ける**（選ばれたら自動進行を止め会話する）。
   - 自分で書く → 共有マスター `~/.claude/travel/templates/profile-template.md` を `~/.claude/travel/profiles/<名前>.md` にコピーし、記入してもらう。
   - 対話で埋める → AskUserQuestion で **1回2問まで**、テンプレの軸を聞く（雰囲気/ペース/食/宿/体力/予算/苦手）。深掘りせず `Other` 自由記述に任せる。
3. `~/.claude/travel/templates/profile-template.md` の構造で Write 保存。最初の観察ログとして「初回ヒアリング」を1件記録。

### モード B: 学習（最重要・このスキルの核心）
**会話・同行者の発言・前フェーズのメモから好みを検知したら、観察ログに追記する。**
- トリガー例: 「〜に行きたいって言ってた」「彼は辛いの苦手」「前回〇〇を気に入ってた」「もっとのんびりしたい」など、好み・要望・反応を示す発言すべて。
- やること:
  1. 対象者のプロファイルを Read（無ければモードAで先に作成）。
  2. 観察ログに1エントリ追記（Edit）。形式:
     ```
     - **2026-06-05** ［文脈: バンコク計画中］ 「夜市でローカルフード食べ歩きしたい」と発言 → 屋台・ローカル体験志向。観光地より生活感のある場所を好む傾向
     ```
  3. 同種の観察が**複数たまったら**「確定プロファイル」へ昇格させ蒸留（Edit）。
     例: 屋台志向の発言が3回 → 確定プロファイルの食欄に「ローカル屋台・食べ歩きを好む」を追加。
  4. 昇格・更新したら一言伝える: 「〇〇さんのプロファイルに『ローカル屋台好き』を学習しました」。
- **provenance を必ず残す**: いつ・どの文脈で・何と言ったか。後で『なぜこの好みと判断したか』を辿れるように。

### モード C: 確認・手動更新
- Read して内容を要約提示。「変えたい/追記したい点は？」と聞き Edit で更新。
- 古くなった・矛盾する好みは観察ログに「更新: 〜は今は当てはまらない」と追記し、確定プロファイルを修正。

## 学習ループの運用ルール（他フェーズと共有）
- basics / combine / refine の最中に好みを示す発言が出たら、**その場でモードBを実行**してプロファイルに書き戻す。
- 1旅行が終わったら（brochure 完成後）、その旅行で得た好み（気に入った/退屈だった等）をまとめて観察ログに追記する。
- 矛盾する観察は消さずに残す（人の好みは変わる）。確定プロファイルだけ最新に保つ。

## 次フェーズへの案内
作成・更新後:
> 〇〇さんのプロファイルを更新しました（`~/.claude/travel/profiles/<名前>.md`）。
> 基本情報を決めるなら `trip-basics`、すでに決まっていて案出しに進むなら `trip-combine` を実行できます。

## 注意
- プロファイルは**好み**。特定旅行の日程・予約は書かない（それは旅行フォルダ側）。
- 同名ファイルは上書きせず必ず Edit で差分更新（蓄積を失わない）。
- 複数人ぶんは1人1ファイル。グループ提案時は全員ぶん Read して共通項と対立項を見る。
- **同行者の“個別”の好みを促す**：複数人をまとめて聞き取ると共通項ばかりになり、同行者個人のプロファイルが薄くなりがち。好みが割れそうな軸（辛さ耐性・体力・絶対やりたいこと・苦手）は**本人別に確認して各ファイルへ蓄積**する。会話で誰か個人の好み（「彼女は〜が苦手」等）が出たら、その都度**その人のファイル**にモードBで追記する（共通ファイルに丸めない）。
